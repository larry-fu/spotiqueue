# Running spotiQueue

## 1. Create a Spotify app

1. Go to the [Spotify Developer Dashboard](https://developer.spotify.com/dashboard) and log in.
2. Click **Create app**.
   - App name / description: anything.
   - Redirect URI: `spotiqueue://redirect` (must match exactly, including the scheme).
   - APIs used: check "Web API".
3. Save, then open the app's **Settings** and copy the **Client ID**.

## 2. Configure the app

Edit `src/config/spotify.ts`:

```ts
export const SPOTIFY_CLIENT_ID = 'YOUR_SPOTIFY_CLIENT_ID'; // paste your Client ID here
```

Leave `SPOTIFY_REDIRECT_URI` as `spotiqueue://redirect` unless you also change the
`scheme` in `app.json` — the two must match.

## 3. Install dependencies

```bash
npm install
```

## 4. Run the app

Custom-scheme OAuth redirects (`spotiqueue://redirect`) are not reliably handled inside
the plain **Expo Go** app, so use a development build:

```bash
# iOS (requires macOS + Xcode)
npx expo run:ios

# Android (requires Android Studio / SDK)
npx expo run:android
```

Both commands build a dev client, install it on a simulator/emulator or connected
device, and start the Metro bundler.

If you only have Expo Go available and want to try it anyway, run `npx expo start`
and open the app in Expo Go — login may work depending on your Expo SDK version, but
if the redirect doesn't return you to the app, switch to a dev build.

## 5. Log in and test

1. Tap **Connect with Spotify** and authorize the app.
2. Search for a song.
3. Tap **Queue** to add it to your active playback queue, or **Playlist** to add it to
   one of your existing playlists.

### Requirements for "Add to Queue"

- Your Spotify account must have **Premium**.
- The Spotify app must have an **active device** — i.e. something already playing (or
  recently playing) on some device — or the queue call will fail with "no active
  device found."

### Requirements for "Add to Playlist"

- Works on any Spotify account (Free or Premium).
- Only playlists you own or have modify access to will accept new tracks; playlists
  owned by other users will show up in the picker but the add call will fail if you
  don't have edit rights.

## Troubleshooting (Linux / Android emulator)

Notes from getting this running on Ubuntu (Wayland + HiDPI, Intel iGPU).

### `:app:configureCMakeDebug` fails with "A restricted method in java.lang.System has been called"

Build must run under **JDK 17**. The system default here is JDK 25, and JDK 21+
makes the Android Gradle Plugin misparse a benign JDK warning (JEP 472) as a fatal
error during the Prefab/CMake step.

Fix: point `JAVA_HOME` at a JDK 17 before running `npx expo run:android`. A copy
already lives at `~/.gradle/jdks/eclipse_adoptium-17-amd64-linux.2` (provisioned by
Gradle). This is set in `~/.bashrc`. Don't put it in `android/gradle.properties` —
that folder is gitignored and regenerated by `expo prebuild`.

### "No Android connected device found, and no emulators could be started automatically"

Expo won't auto-create an emulator. Create/launch one yourself, then run Expo:

```bash
# one-time: create an AVD (needs a system image installed via sdkmanager first)
avdmanager create avd -n Pixel_7 -k "system-images;android-36.1;google_apis;x86_64" -d pixel_7

# launch it (hardware GPU accel — much faster than the default software renderer)
emulator -avd Pixel_7 -gpu host -no-audio
```

`emulator` lives in `$ANDROID_HOME/emulator`. `adb` only talks to already-running
devices; it does not start emulators. Ignore the stray early log line
`ERROR | Your GPU cannot be used for hardware rendering` — it's a false X11 probe and
the emulator still picks the Intel GPU via Vulkan.

### Emulator sidebar toolbar Home/Back/Overview buttons do nothing

Known flaky on this setup (only the "..." extended-controls button works). The guest
also defaults to gesture navigation, so there are no on-screen buttons inside the
device either. Switch the guest to a 3-button nav bar (persists across restarts):

```bash
adb shell cmd overlay enable-exclusive --category com.android.internal.systemui.navbar.threebutton
```

`adb shell input keyevent KEYCODE_HOME` (or `KEYCODE_BACK` / `KEYCODE_APP_SWITCH`)
also works anytime, bypassing the toolbar entirely.

### Physical keyboard doesn't type into text fields

The AVD is created with `hw.keyboard=no`, so the host keyboard isn't forwarded to the
guest (only the on-screen soft keyboard works). Set `hw.keyboard=yes` in
`~/.android/avd/Pixel_7_2.avd/config.ini` and restart the emulator. You can also type
into a focused field over adb: `adb shell input text "hello"`.
